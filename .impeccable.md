## Users

**Público-alvo:** Clientes de design e desenvolvimento (agências, startups, empresas buscando talento técnico).

**Contexto de uso:**
- Acessando via desktop durante processo de avaliação de portfólio
- Querem ver não só o *que* foi feito, mas *como* e *com qual qualidade*
- Buscam evidências de: estética refinada, atenção ao detalhe, domínio técnico, capacidade de entregar experiências diferenciadas
- Tempo limitado — precisam ser "conquistados" nos primeiros 10 segundos

**Job to be done:**
> "Preciso decidir se esse dev/designer é o parceiro certo para meu projeto. Ele tem o nível técnico e o olho estético que preciso?"

## Aesthetic Direction

**Referência primária:** [Zentry](https://zentry.com) — experiências imersivas, storytelling visual, interações que recompensam exploração

**Visual tone:**
- Dark mode exclusivo (preto absoluto como canvas)
- Elevação através de luz e sombra (violeta como accent estratégico)
- Espaço negativo generoso — respiração entre elementos
- Micro-interações que deliciam (hover states, transições fluidas)
- 3D usado com propósito, não como gimmick

**Anti-referência:** Templates genéricos de Bootstrap — evitar a qualquer custo o visual "mais do mesmo"

**Modo:** Dark-only (sem light mode — parte da identidade)

## Design Principles

### 1. **Experiência Imersiva Primeiro**
Cada interação deve recompensar o usuário. Scroll não é apenas navegação — é storytelling. Animações não são decoração — são comunicação.

### 2. **Dark como Canvas, Luz como Pincel**
O preto (#000) não é ausência — é o fundo que faz os elementos brilharem. Use o violeta (#a855f7) com intenção: nunca mais de 10% da tela, mas sempre onde a atenção deve ir.

### 3. **Tecnologia Visível**
Não esconda a stack. React Three Fiber, Framer Motion, WebGL — essas são ferramentas de prestígio. Deixe claro que você domina o que há de mais moderno.

### 4. **Performance como Estética**
60fps não é negociável. Um efeito que dropa frames é pior que nenhum efeito. Prefira menos animações perfeitas do que muitas quebradas.

### 5. **Acessibilidade Inclusiva**
Dark mode e animações não são desculpas para exclusão. `prefers-reduced-motion`, focus visible, contraste adequado — são requisitos, não luxos.

## Checklist de Qualidade

Antes de entregar qualquer componente:
- [ ] 60fps garantido em animações
- [ ] Hover states em elementos interativos
- [ ] Focus visible para navegação por teclado
- [ ] `prefers-reduced-motion` respeitado
- [ ] Contraste WCAG 2.1 AA mínimo
- [ ] Mobile-first (mas desktop é prioridade para esse público)
